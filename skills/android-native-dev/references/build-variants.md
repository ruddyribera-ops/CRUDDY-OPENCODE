# Build Variants & Product Flavors

Product Flavors allow different versions of your app (e.g., free/paid, dev/staging/prod).

## Configuration in app/build.gradle.kts

```kotlin
android {
    flavorDimensions += "environment"

    productFlavors {
        create("dev") {
            dimension = "environment"
            applicationIdSuffix = ".dev"
            versionNameSuffix = "-dev"
            buildConfigField("String", "API_BASE_URL", "\"https://dev-api.example.com\"")
            buildConfigField("Boolean", "ENABLE_LOGGING", "true")
            resValue("string", "app_name", "MyApp Dev")
        }
        create("staging") {
            dimension = "environment"
            applicationIdSuffix = ".staging"
            versionNameSuffix = "-staging"
            buildConfigField("String", "API_BASE_URL", "\"https://staging-api.example.com\"")
            buildConfigField("Boolean", "ENABLE_LOGGING", "true")
            resValue("string", "app_name", "MyApp Staging")
        }
        create("prod") {
            dimension = "environment"
            buildConfigField("String", "API_BASE_URL", "\"https://api.example.com\"")
            buildConfigField("Boolean", "ENABLE_LOGGING", "false")
            resValue("string", "app_name", "MyApp")
        }
    }

    buildTypes {
        debug { isDebuggable = true; isMinifyEnabled = false }
        release {
            isDebuggable = false; isMinifyEnabled = true
            proguardFiles(getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro")
        }
    }
}
```

**Build Variant Naming:** `{flavor}{BuildType}` → e.g., `devDebug`, `prodRelease`

## Gradle Build Commands

```bash
# List all available build variants
./gradlew tasks --group="build"

# Build specific variant
./gradlew assembleDevDebug
./gradlew assembleStagingDebug
./gradlew assembleProdRelease

# Build all variants of a flavor
./gradlew assembleDev       # All Dev variants
./gradlew assembleProd      # All Prod variants

# Build all variants of a build type
./gradlew assembleDebug     # All flavors, Debug build
./gradlew assembleRelease   # All flavors, Release build

# Install variant to device
./gradlew installDevDebug
./gradlew installProdRelease
```

## Access BuildConfig in Code

> **Note**: Starting from AGP 8.0, `BuildConfig` is not generated by default. Enable it:
> ```kotlin
> android { buildFeatures { buildConfig = true } }
> ```

```kotlin
val apiUrl = BuildConfig.API_BASE_URL
val isLoggingEnabled = BuildConfig.ENABLE_LOGGING
if (BuildConfig.DEBUG) { /* Debug-only code */ }
```

## Flavor-Specific Source Sets

```
app/src/
├── main/           # Shared code for all flavors
├── dev/            # Dev-only code and resources
├── staging/        # Staging-only
├── prod/           # Prod-only
├── debug/          # Debug build type code
└── release/        # Release build type code
```

## Multiple Flavor Dimensions

```kotlin
android {
    flavorDimensions += listOf("environment", "tier")
    productFlavors {
        create("dev") { dimension = "environment" }
        create("prod") { dimension = "environment" }
        create("free") { dimension = "tier" }
        create("paid") { dimension = "tier" }
    }
}
// Results in: devFreeDebug, devPaidDebug, prodFreeRelease, etc.
```
