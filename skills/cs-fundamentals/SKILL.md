# CS Fundamentals

Use this skill when explaining technical concepts, tradeoffs, algorithms, data structures, or systems behavior.

## Rules
- Start from the user's concrete code or question when available.
- Define terms plainly before using them in deeper analysis.
- Include time and space complexity when algorithmic behavior matters.
- Prefer small examples over abstract lectures.
- Call out practical failure modes and debugging checks.
